The Capstone · Build Plan

Build a portfolio that proves the pitch — on Nuxt 4.

Eighty teardowns of the best sites on the web converge on one lesson: world-class is restraint plus one flawless signature move. This is the concrete plan to ship that, on a Vue 3 + TypeScript stack you can actually maintain.

01 / Vision

Vision & positioning

A top-tier developer portfolio has exactly three jobs, in order: establish credibility in the first second, make the work legible and specific, and make contacting you frictionless. Everything else is decoration. Across the 80 teardowns the failure mode is always the same — sites that scatter ten half-finished effects read as junior; sites that execute one idea flawlessly read as senior.

The meta-pattern, stated once: restraint reads as seniority; one signature move reads as craft; specificity reads as competence. Spectacle without those three is noise.

The chosen direction: restrained editorial + one signature interactive moment

There are four defensible archetypes in the data. We pick deliberately, and reject the rest for this project:

ArchetypeExemplarsVerdict for you
Calm editorial engineer Brittany Chiang, Sara Soueidan, Lee Robinson, Pol Piella Adopt as the base. Highest signal-to-effort, ages well, a11y-friendly, maintainable solo.
The site IS the demo reel Cassie Evans, Josh Comeau, Bruno Simon Borrow the principle (prove, don't claim) via ONE moment, not a whole WebGL site.
Full WebGL / 3D world Henry Heffernan, Tim Roussilhe, Lusion Reject as the whole site. Huge build + maintenance cost, real a11y/SEO tax (see §7).
Radical prose-first Lee Robinson, Manu Arora, Olaolu Only wins if your writing carries it. Mine for confidence, not as the template.

The direction: a near-monochrome, type-led editorial layout (the calm-engineer base) with exactly one concentrated interactive moment that performs your craft — a generative or motion-driven hero set piece — and disciplined micro-interactions everywhere else. This is the "Cassie/Josh principle on a Chiang skeleton."

Why this is defensible for you specifically: you are a full-stack developer, not a 3D specialist, shipping a static Nuxt 4 site to GitHub Pages workings/SPEC. A drivable-car WebGL world (Bruno Simon) would be a six-month project you can't maintain; a Hugo template (Caleb Brown) is the floor. The editorial-plus-one-moment direction is the only one that is simultaneously achievable solo, maintainable, fast on a static host, and still memorable. It maximizes craft-per-line-of-code.

03 / Architecture

Information architecture & content plan

Single-page-first, with content-driven sub-routes only where the writing earns them. The Nuxt 4 SPEC workings/SPEC already commits to a hero + content-driven section; this expands that into a complete IA without bloating the build.

Page / section map

SectionWhat it doesWhat to show
Hero Credibility in 1 second + the signature moment Name, one-line role thesis ("Full-stack developer who helps clients ship applications"), the ONE interactive set piece, a primary CTA.
About The human + the proof 3–5 sentences of voice (Lee/Sara register), current focus, a "now" line. Authored in content/about.md.
Selected work The portfolio proper 3–6 case studies, numbered (01–06), year-tagged, mono metadata. Real screenshots / live links, never abstract placeholders.
Experience / writing Specificity = competence Dated list of roles, talks, shipped apps with named companies + quantified impact. Optional blog index if you write.
Contact Frictionless reach The 3 existing icon links (Instagram, LinkedIn, email) + a "copy email" tell. No cold form where a one-liner works.

Narrative arc

Hero (who + a spark) → About (why trust you) → Work (the evidence) → Experience (the receipts) → Contact (the close). Each section is one scroll-spy node in the right column of the Chiang layout. Wayfinding is editorial: numbered grids, NN / total counters, year tags, monospace metadata — cheap, high-signal, and on-brand for a dev audience.

Project case-study template — author each as one content/work/<slug>.md with frontmatter. Render with <ContentRenderer>. Fields:

---
title: "Realtime Chat App"
slug: "chat-app"
year: 2025
role: "Full-stack · solo"
stack: ["Vue 3", "WebSocket", "Postgres"]
cover: "/work/chat-app/cover.webp"
live: "https://…"
repo: "https://github.com/…"
order: 1
---

**Problem** — one sentence on what was broken / needed.
**Approach** — 2–3 sentences, the key technical decision (and the trade-off).
**Outcome** — quantified: "cut p95 latency 280ms→40ms", "1.2k MAU".
**Artifact** — a real screenshot or an embedded live demo, not a mockup.

Keep it to four beats: Problem / Approach / Outcome / Artifact. Specificity over adjectives — "passionate developer" is worth nothing; "cut latency 7×" is worth everything.

04 / Design system

Design system & tokens

One face for display, system stack for body, one accent. The discipline matters more than the values. Define tokens once as CSS custom properties (or Tailwind v4 @theme, since the SPEC uses @nuxt/ui 4 on Tailwind v4) and never hardcode again. See Typography, Color, and Styling.

Type scale

Fluid clamp() scale, hierarchy from weight + size, not from many sizes. Your existing NeuzeitGro + Quicksand stack carries over workings/SPEC — treat NeuzeitGro as the one characterful display face doing the brand work; keep the body stack near-system for payload.

/* tokens.css — type */
--font-display: "NeuzeitGro", Georgia, serif;   /* hero + headings */
--font-body:    "Quicksand", system-ui, sans-serif;
--font-mono:    "JetBrains Mono", ui-monospace, monospace; /* metadata */

--step--1: clamp(0.80rem, 0.76rem + 0.18vw, 0.875rem);
--step-0:  clamp(1.00rem, 0.95rem + 0.22vw, 1.125rem);  /* body */
--step-3:  clamp(1.80rem, 1.45rem + 1.70vw, 3.05rem);   /* section h */
--step-4:  clamp(2.30rem, 1.70rem + 2.90vw, 4.40rem);   /* hero */

Apply aggressive negative tracking on the oversized hero (letter-spacing: -0.025em) — the recurring "premium type" tell across Mercury, Vercel, Robin Noguier.

Color

Near-monochrome base + one accent. Your current cosmic/dark-orchid background gives you a natural dark base; pick a single accent that survives both modes (dark-mode toggle is a real SPEC feature). Light-mode and dark-mode are two token sets behind @nuxtjs/color-mode.

--bg: #0b0c0e; --surface: #16191e; --line: #23272e;
--ink: #e9eaec; --ink-soft: #b7bcc4; --ink-dim: #888f9a;
--accent: #f5a623;            /* exactly one — see Dept/Ramp/Nike */
/* AA contrast is non-negotiable; verify every pair (see §7) */

Spacing & motion tokens

--s-1:.25rem --s-2:.5rem --s-3:.75rem --s-4:1rem --s-5:1.5rem --s-6:2rem --s-7:3rem
--r-2:8px --r-3:12px --r-pill:999px
--ease: cubic-bezier(0.22, 0.61, 0.36, 1);   /* one signature easing */
--dur-fast: 180ms; --dur-base: 320ms; --dur-slow: 600ms;

One easing curve used everywhere is itself a coherence signal (Andrei Terteci's "one gradient" lesson, applied to motion).

05 / Motion

Motion & interaction budget

The single highest-leverage decision in the whole build: spend the wow on ONE thing, stay disciplined everywhere else. See Motion, Animation, Interaction.

The one signature moment — pick A or B

A · Particle / generative hero (low-risk)

Evolve the existing @tsparticles/vue3 cobweb into the signature — cursor-reactive links that bend toward the pointer, a tasteful constellation that responds. You already have the dependency in the SPEC; this is incremental, degrades gracefully, and stays cheap. Recommended starting point.

B · Scroll-driven hero set piece (higher ceiling)

Lenis smooth scroll + GSAP ScrollTrigger pinning a hero that transforms as you scroll (à la Apple's scrub, scaled down). Higher craft ceiling, higher build + perf + a11y cost. Choose only if you'll invest the polish — a half-done scroll effect is worse than none.

Restraint everywhere else — the cheap, disproportionate wins

  • Magnetic nav links/buttons (Dennis) — ~20 lines, premium feel. workings/dennis-snellenberg
  • Copy-email tell (Rauno) — button flips to "Copied ✓". workings/rauno-freiberg
  • Scroll-spy active-section nav (Chiang) — IntersectionObserver, no library.
  • View Transitions for route changes — Nuxt 4 has built-in support (see §6); eliminates navigation flash for near-zero code.
  • Section reveal on scroll — a single fade-up with @vueuse/motion, one easing, short distance. No staggered confetti.

Perf & reduced-motion guardrails (non-negotiable):

  • Wrap every non-essential animation in @media (prefers-reduced-motion: reduce) → reduce to opacity-only or none. Lenis must be disabled under reduced-motion.
  • Particles/WebGL only inside <ClientOnly>, lazy-mounted, paused when offscreen / tab hidden (visibilitychange).
  • Animate only transform and opacity (compositor-friendly). Never animate layout properties.
  • Budget: the signature moment must not push INP > 200ms or drop the hero below 60fps on a mid-tier laptop.
06 / Tech

Tech architecture — Nuxt 4 specifics

Stack is fixed by the rebuild SPEC workings/SPEC: Nuxt 4.4 (app/ dir), full TypeScript, @nuxt/ui 4 (Tailwind v4), Pinia 3, @nuxt/content 3, @nuxt/fonts, @vite-pwa/nuxt, Vitest 4, pnpm, static export to GitHub Pages. The plan below maps the design decisions onto that stack. See Meta-Framework and State.

Rendering strategy

SSG / prerender. ssr: true + full prerender → static HTML to .output/public, deployed to gareth0712.github.io. This is correct: a portfolio is content-stable, and static-first is the universal norm across the dev-portfolio teardowns (Astro / Eleventy / Next export recur constantly). Fast first paint is treated as a feature.

// nuxt.config.ts
export default defineNuxtConfig({
  ssr: true,
  nitro: { prerender: { crawlLinks: true, routes: ['/'] } },
  experimental: { viewTransition: true },   // page-morph, near-zero code
  modules: [
    '@nuxt/ui', '@nuxt/content', '@nuxt/fonts',
    '@pinia/nuxt', '@vite-pwa/nuxt',
  ],
})

Content (Nuxt Content 3)

About + each case study live in content/ as markdown, queried via queryCollection and rendered with <ContentRenderer>. Define a typed collection in content.config.ts so frontmatter is type-safe — work items get the schema from §3.

Image & font optimization

  • Fonts: @nuxt/fonts for Quicksand (auto self-host + preload + font-display: swap); NeuzeitGro carried over as a local @font-face .ttf — subset it to Latin and preload only the hero weight to protect LCP.
  • Images: add @nuxt/image for case-study covers → WebP/AVIF, responsive srcset, lazy below the fold, explicit width/height to kill CLS. The cosmic background should be a sized, compressed asset, not a raw JPG.

Motion libraries

NeedPickNote
Smooth scrollLenis (only if signature B)Disable under reduced-motion. Skip entirely if you choose signature A.
Scroll-scrub / timelineGSAP + ScrollTriggerOnly for signature B. Heavyweight — justify it.
Reveal / micro-motion@vueuse/motionVue-native, tiny, declarative v-motion. Default choice for the 90% case.
Page transitionsNuxt View TransitionsBuilt-in; no extra dependency.

State (Pinia 3)

Keep it minimal — a portfolio barely needs global state. One useThemeStore for the dark-mode preference (bound to @nuxtjs/color-mode, persisted), per the SPEC. Resist adding stores for things that are local component state. See State.

Next.js equivalent (since the plan is meant to be reusable): Nuxt SSG → Next output: 'export' / SSG; Nuxt Content → next-mdx-remote or Contentlayer; @nuxt/image → next/image; @nuxt/fonts → next/font; Pinia → Zustand; View Transitions → the Next App Router View Transitions API; @vueuse/motion → Framer Motion. The architecture is identical; only the bindings change.

07 / Budget

Performance & accessibility budget

Core Web Vitals targets

MetricTargetHow
LCP< 1.5s (good: < 2.5s)Static HTML, preloaded hero font, sized hero image, no render-blocking JS.
INP< 200msKeep main thread free; lazy-mount the signature moment; debounce magnetic/scroll handlers.
CLS< 0.1Explicit width/height on all media; font-display: swap with a matched fallback metric.
JS payload< 120KB gz initialNo GSAP/Lenis unless signature B; tree-shake; particles in a deferred chunk.
Lighthouse95+ all fourRun in CI; treat a regression as a build failure.

Accessibility checklist

  • Skip-link as the first focusable DOM node (Sara/Chiang). Real landmarks: header / main / nav / footer.
  • Every interactive element keyboard-reachable with a visible :focus-visible ring. Magnetic effects must not trap or hide focus.
  • AA contrast on every text/background pair, in both themes — verify, don't assume.
  • Links distinguishable without color (underline or weight), not color alone.
  • Respect prefers-reduced-motion globally (§5). Canvas/particles get an aria-hidden + a real text DOM underneath.
  • Alt text on every meaningful image; decorative covers alt="".
  • Semantic heading order (one h1, no skipped levels) — also your SEO backbone.

The honest trade-off we saw across the teardowns. WebGL-heavy sites (Bruno Simon, Henry Heffernan, Tim Roussilhe, Lusion) are stunning but pay a real tax: weak SEO (content lives in a canvas crawlers can't read), fragile a11y (you must hand-build a parallel DOM, as Henry does), heavy payloads, and ongoing maintenance. workings/henry-heffernan The calm-engineer base sidesteps all of it — your content is real HTML, indexable and accessible by default, and the one signature moment is an enhancement layered on top, never the load-bearing structure. That's the whole reason for the chosen direction in §1.

08 / Roadmap

Phased build roadmap

Three phases, shippable at the end of each. Don't build the signature moment first — get a fast, accessible, complete site live, then layer the wow. A half-built portfolio with a fancy hero is worse than a complete plain one.

Phase 1 · MVP — complete, fast, accessible (the foundation)

  • Scaffold Nuxt 4 + TS + tokens (§4) per the SPEC's T1/T2. Migrate fonts, background, favicons.
  • Build the Chiang split-screen shell: sticky identity rail + scrolling content column.
  • All five sections from §3 with real content — About + ≥3 case studies authored in content/.
  • Dark-mode toggle (real, persisted) + the 3 contact links with copy-email tell.
  • Hit the §7 budget: Lighthouse 95+, AA contrast both themes, skip-link, semantic landmarks.
  • Deploy static to GitHub Pages via the fixed Actions workflow. Ship it.

Phase 2 · The signature moment

  • Pick A (cursor-reactive particle hero) or B (scroll-scrub set piece) from §5 — A recommended.
  • Lazy-mount inside <ClientOnly>; pause offscreen / tab-hidden; full reduced-motion fallback.
  • Re-verify the budget — the moment must not regress LCP/INP/CLS or 60fps.

Phase 3 · Polish — the micro-interaction layer

  • Magnetic nav links/buttons (Dennis). Scroll-spy active-section highlight (Chiang).
  • Nuxt View Transitions on route changes. @vueuse/motion section reveals, one easing.
  • One human touch — a signature SVG, an easter egg, a live "now" line (Jhey).
  • Final pass: cross-browser, mobile, Lighthouse, axe a11y audit, real-device check.

The one sentence to remember. Build the calm, fast, accessible editorial site first — that alone beats 90% of dev portfolios — then spend your entire motion budget on a single moment that proves you can do the thing you say you can do. Restraint, one signature, specificity. Everything in this plan serves those three.
